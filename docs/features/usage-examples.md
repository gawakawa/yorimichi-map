# 💡 使用例とデモシナリオ

## 基本的な使い方

### デモシナリオ1: スポット経由のルート作成

**ユーザー**: 「東京駅から箱根湯本まで行きたい。途中で評価の高いお蕎麦屋さんに寄りたい。」

**AIの応答**:

1. ルート沿いの高評価な蕎麦屋を自動検索
2. 最適な経由地を含むルートを計算
3. インタラクティブな地図でルートプレビュー表示
4. 予想時間、距離、通行料金を表示

**ユーザー**: 「完璧！その蕎麦屋さん経由で行こう。」

**アクション**: 「🚀 Googleマップでナビ開始」ボタンをクリック

- Googleマップアプリが自動起動
- 経由地を含むルートがすでに設定済み
- すぐにナビゲーション開始可能

**帰路作成**: 「🔄 帰りのルートを作成」ボタンをクリック

- AIが自動的に帰路ルートを生成
- 経由地を逆順にした最適な帰り道を提案

## デモシナリオ（ハッカソン用）

### シナリオ1: 基本的な使い方

**ユーザー**: 「東京駅から箱根湯本まで行きたい。途中で評価の高いお蕎麦屋さんに行きたい。」

**AI**:

- 自動で`search_places`で蕎麦屋を検索
- `calculate_route`でルート計算
- 地図が表示される

**ユーザー**: 「ありがとう。じゃあその蕎麦屋さん経由で。」

**アクション**:

- 右側の「🚀 Googleマップアプリでナビ開始」ボタンをクリック
- 実際のGoogleマップアプリが開く

### シナリオ2: 帰路の自動生成

**アクション**: 「🔄 経由地を逆にして帰宅ルートを提案」ボタンをクリック

**AI**:

- 自動的に出発地と目的地を入れ替え
- 経由地を逆順にしたルートを計算
- 地図を更新

## 会話例

### 例1: カジュアルなリクエスト

**ユーザー**: 「お腹すいた。近くでラーメン食べたい。」

**AI**: 「現在のルート周辺で評価の高いラーメン店を検索しますね！」

- Places APIで検索
- 評価4.0以上の店を3件提案
- 各店の特徴を説明

**ユーザー**: 「2番目のお店がいいな。」

**AI**: 「わかりました！そのお店経由のルートを計算します。」

- ルートを再計算
- 地図を更新

### 例2: 景観重視のリクエスト

**ユーザー**: 「海沿いの道で行きたい。景色がいいところ。」

**AI**: 「海沿いのルートを優先して計算しますね。」

- Routes APIでルート計算
- 複数のルートオプションを提案
- 所要時間と料金を比較

### 例3: 複数経由地

**ユーザー**: 「温泉に入ってから、美術館も寄りたい。」

**AI**:

1. 温泉施設を検索
2. 美術館を検索
3. 最適な順序でルートを提案
4. 所要時間と料金を表示

## 使用上のヒント

### より良い結果を得るために

1. **具体的な場所名**: 「東京」より「東京駅」の方が正確
2. **要望を明確に**: 「評価が高い」「駐車場がある」など
3. **時間帯の指定**: 「昼食」「ディナー」など
4. **予算の指定**: 「リーズナブルな」「高級な」など

### 対応可能なリクエスト

- 飲食店の検索（「ラーメン」「カフェ」「蕎麦」など）
- 観光地の検索（「美術館」「温泉」「展望台」など）
- ルート条件（「高速道路優先」「一般道」「料金安く」など）
- 帰路の作成（「帰りのルート」「同じ経由地で帰りたい」など）

### 制限事項

- 経由地は最大8箇所まで
- リアルタイム交通情報は出発時刻から5分後を想定
- Places APIは評価4.0以上のスポットを優先
- 1回の検索で最大3件のスポットを提案
